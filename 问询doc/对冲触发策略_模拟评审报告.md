## 0. 前置声明（必须先读）

这是一份**控制策略（control policy）的工程分析**，不是行情预测，也不是投资建议（本模型非持牌投顾）。请按以下边界理解全部数字：

1. **模型相关。** 全部结论建立在 7 个人工路径生成器之上，它们是一个"压力实验室"，不是对 BTC 未来分布的预测。表中的绝对 USD 数值**只能用于横向比较不同 policy**，不能当作实盘 PnL 目标。
2. **"均值收益"被压力集高估。** 多数集成里 Whipsaw/Trend/Intraday 这类强制不利场景占比偏高，会把"对冲改善均值"夸大。**诚实的读法是:在贴近真实的 calm 集成上,对冲是"近乎零均值成本的尾部保险",而不是"对冲能赚钱"。**
3. **必须用 FMZ 实盘日志校准后才能上线。** 一个正向信号:你给的 2026-06-28 06:04:47 真实成交(限价 SELL≈1380 USDC,fee 0.552 ≈ 4bps taker)与本模型的 taker_fee=4bps 完全吻合;但临近交割的滑点、真实盘口深度、真实 expiry-gamma 仍需实盘数据。
4. **遵守文档硬边界。** 下文所有建议**不新增任何 FMZ 运行时对冲指令**;FMZ 唯一交互入口仍是计划确认码(`执行:<码>` / `EXECUTE:<码>` / 裸码),`POSITION_MANAGE` 仍是非交互只读屏。候选库、确认码、入场确认、常规止盈、风险退出预算逻辑**一律不动**。本方案是对**已存在的 v3.1.3 对冲控制器**做 sizing/执行/节奏的内部重构,不是新交互面。

## 1. 执行结论（一句话 + 取舍）

> **保留(KEEP)"风险恶化 → 引入 Perp 对冲"的哲学,但要收紧(TIGHTEN),不要放松、不要替换。**

**为什么是 KEEP:** 在**每一个**集成、**每一组**参数下,"做一点对冲"在 CVaR5(尾部)上都优于"完全不对冲";在贴近真实的 calm+stress 混合上,它还能改善或几乎不损伤均值。触发式对冲方向正确。

**为什么是 TIGHTEN 而非"维持现状":** 当前的 breach-probability 策略(`1_Current_v314`)留下大量改进空间,且**在 IV<RV / whipsaw 区间会反向伤害**(见 §3 表6:CVaR5 -882 vs 不对冲 -732,均值 -166 vs -144)。维持原样不可取。

**为什么不是 LOOSEN(更少/更晚对冲):** `2_Delayed`(延迟首次对冲)和 `3_VolSpeed`(vol-speed AND 门控)都更差 —— 其中 `3_VolSpeed` 门控过严,几乎不开仓,`tail_improve` 甚至为**负**(-49),即"偶尔在错误时点对冲却没保住真正的尾部",比不对冲还糟。

**为什么不是 REPLACE(推翻触发式、改全程 delta-neutral 或纯 ML):** 不符合文档约束,也无必要。下文的 gamma-aware 是对**现有触发逻辑的收紧**,不是替换。

**收紧的 4 个动作(全部 CONFIG 可独立开关、业务主链不动):**

| # | 动作 | 一句话理由 | 量化收益(ALL 集成) |
|---|------|-----------|---------------------|
| ① | **Sizing: breach-prob 分档 → gamma-aware(美元 gamma 缩放)** | 把对冲集中到"真正危险处"(近行权价/近到期/高 delta-speed),远离行权价的虚惊不再过度对冲 | 仅此一项 均值 -82→-47、CVaR5 -594→-480 |
| ② | **Execution: reduce 腿 maker-first** | 减仓不紧急(价格已朝有利方向走),先挂 maker,不成交再吃单 | 成本 $73→$57、均值 -82→-66 |
| ③ | **Hold-the-hedge + no-trade band** | min_hold 12min + reduce 冷却 + rebalance band=满仓 20%,直接干掉微抖动(churn 2.9→0.5,trades 9.6→5.8) | 这是你最担心的 whipsaw 成本的主解 |
| ④ | **保留 crash-override** | 极端 price-speed 旁路门控,gap-down 的便宜保险,final-3h 也常开 | 单独成本极低 |

三者叠加 = **BEST 候选:均值 -39、CVaR5 -468、CVaR1 -677、churn 0.5、成本 $38**,在所有指标上同时优于 Current 与单纯 GammaAware。

**明确否决:** vol-speed AND 门控(主动有害)、first-hedge-delay(均值与尾部双损)。

## 2. 模拟设计

**标的与组合:** 短 PUT 或短 CALL 垂直价差,默认 base case = S0=60000,DTE=24h,短腿初始 |delta|=0.25,保护腿宽 W=2000 USD,IV=45%,RV=35%,1 BTC notional(1 contract=1 BTC,USD 计 PnL,r=0,365 日年化)。

**7 个路径生成器(GENERATORS):**

| 类别 | 生成器 | 用途 |
|------|--------|------|
| **Calm(≈无漂移,贴近真实无条件分布)** | `GBM` | 平滑基线,无跳跃 |
| | `Jump`(Merton,带补偿器净漂移≈0) | 跳跃尾部 |
| | `Heston`(随机波动,ρ=-0.6) | 波动聚集 |
| | `Regime`(4 态 Markov:low/normal/high/crash) | 状态切换 |
| **Stress(条件式不利压力测试)** | `Whipsaw`(强制下行后反转) | 制造 round-trip 抽打 |
| | `Trend`(强制下行后续行) | 持续单边不利 |
| | `Intraday`(U 型 + 事件 burst) | 日内冲击 |

`_iv_path` 叠加杠杆效应(不利/下行时 IV 抬升,β=1.4)。

**precompute** 对每条路径每步算出:期权 MTM、组合 delta(>0 看多)、组合 gamma(<0 short)、breach 概率 N(-d2)、price-speed(≈10min)、短窗 realized vol(≈20min)、到行权价/盈亏平衡距离。

**run_policy(单一可配置控制器)** 内含:hysteresis(p_soft 启动、p_reduce 清除)、SOFT 持续→满仓升级、add/reduce 冷却、min_hold、**rebalance no-trade band(满仓的 band_frac,抑制微抖动的关键)**、止盈(DTE>3h 时 80% credit)、风险退出(预算 = min(3×credit, 0.5×max_loss),作用于期权腿 MTM)、final-3h 模式(normal/suppress_add/aggressive)。对冲 MTM = Σ h·ΔS,成本按 bps 计。

**成本模型(bps of notional,已用你的实盘 fill 校准):** taker 4 / maker 1 / half_spread 1 / slip 2(crash 时 8),maker 成交概率 0.75。

**集成定义(关键):**
- **Calm 集成** = GBM+Jump+Heston+Regime(≈无漂移,作为贴近真实的无条件参考)
- **Stress 集成** = Whipsaw+Trend+Intraday(条件式不利压力)
- **ALL 集成** = 7 个全池化(stress 偏重,均值偏悲观,仅供横向排序)
- Sweep 用 6 模型混合(GBM/Jump/Heston/Regime/Whipsaw/Trend)以平衡速度与覆盖

**调试已完成的 3 个 bug:** Jump 过度看跌(已重标定到净漂移≈0);连续 delta-reconcile 造成的 92 trades/50 churn 不合理(已加 no-trade band);对冲后最差路径超过期权 max-loss(部分是过度交易,band 修复后趋于合理,残留即真实 whipsaw 成本)。修复后:driftless 路径已验证、24h 内 trades 5.8–9.6、churn 0.5–2.9,量级合理。

| BEST + aggressive | -39.9 | -468.8 | -675.2 | -2052.0 | 5.9 | 0.6 | 38.4 |

**读法:** 三者几乎打平(均值差 ~3 USD,CVaR1 差 ~15 USD)。`suppress_add` 略改善均值与 churn,代价是深尾(CVaR1)略差;`aggressive` 略改善深尾,代价是均值与 churn 略升。→ **推荐默认 `suppress_add`**:最后 3 小时**不再开新对冲腿**(临近交割 gamma 暴涨,新开腿易被交割期噪声抽打),但**保留 crash-override(真崩仍对冲)+ 允许 reduce/unwind**。这与你"24h 末 3h 不主动止盈、放任到交割"的设计哲学一致 —— 末段只管理已有 exposure,不主动新建。

### 表 9 — notional sweep:对冲成本占 credit 比例（BEST,警示用）

| notional(BTC) | credit(USD) | 对冲成本(USD) | **成本/credit%** | trades | CVaR5/单位 |
|---:|---:|---:|---:|---:|---:|
| 1.000 | 205.1 | 37.0 | **18.0%** | 5.8 | -437.6 |
| 0.050 | 10.3 | 3.1 | **30.3%** | 8.8 | -541.8 |
| 0.020 | 4.1 | 1.2 | 30.3% | 8.8 | -543.1 |
| 0.010 | 2.1 | 0.6 | 29.6% | 7.5 | -541.6 |

**读法(重要警示):** notional 越小,成本/credit% **越高**(18%→30%)、单位尾部保护**越差**。原因:Binance 的 **0.001 BTC 最小下单 / 0.1 tick** 是固定颗粒度,在小仓位上变成相对很大的最小步长,使对冲变得"块状",成本占比抬升、保护变粗糙。→ **不要用小额实盘测试的经济性去外推全仓** —— 对冲在全仓位上更划算。小仓位的真实经济性必须在 FMZ 实盘按真实 min/tick 验证。

## 4. BEST 候选策略（完整定义）

**一句话:** gamma-aware sizing(主因子)+ maker-first reduce(成本)+ hold-the-hedge/no-trade band(抗 whipsaw)+ crash-override(尾部保险),final-3h 用 suppress_add。

**核心 sizing 机制(与代码一致):**
将对冲规模从"纯 breach-prob 分档(soft 50% / full 100%)"改为"与归一化美元 gamma 挂钩":

```
gnorm = clip( |组合gamma| · S² / 1e6 , 0, 1 )      # 美元-gamma 归一化
gfrac = clip( 0.30 + 0.70 · gnorm , 0, 1 )          # 启用时底仓30%,近行权价/到期升至满仓
frac  = max( 分档比例 , gfrac )   （仅在 soft_region 或 is_hard 且 active&gate 时）
h_target = - frac · max(组合delta, 0)               # 卖 perp 中和看多 delta
```

含义:**远离行权价(gamma 小)只维持约 30% 底仓 → 避免远端虚惊的过度对冲与 drag;临近行权价/到期(gamma 大)自动升至满仓 → 抓住真正的尾部威胁。**

**完整参数(首版固定,不含 ML):**

| 维度 | 参数 | 值 |
|------|------|---|
| **触发(engage)** | p_soft(breach 概率 N(-d2)) | 0.35 |
| | hard 即时满仓 | p_hard=0.55,或 S ≤ breakeven |
| | crash 旁路 | price-speed ≤ -1.1%(≈10min 窗) |
| **升级** | SOFT 持续→满仓 | persist=8 min |
| | 初始 SOFT 分仓(gamma 之下的底) | soft_frac=0.40 |
| **Sizing** | gamma-aware | gfrac=0.30+0.70·clip(γ·S²/1e6,0,1) |
| **退出/解除** | hysteresis 清除 | p_reduce=0.24(< engage 0.35,防 flip-flop) |
| | 最短持有 | min_hold=12 min |
| | reduce 冷却 / add 冷却 | 4 min / 4 min |
| | rebalance no-trade band | 满仓的 20% |
| **执行** | reduce 腿 | maker-first(maker_fill_prob 0.75,不成交吃单) |
| **末段** | final-3h | suppress_add(保留 crash + reduce/unwind) |

**成本假设(可校准):** taker 4bps / maker 1bps / half-spread 1bps / slip 2bps(crash 8bps)。

## 5. 参数建议（针对文档逐问）

- **Q:何时触发(trigger)?** breach 概率 ≥ 0.35 **或** 现价 ≤ breakeven **或** price-speed ≤ -1.1%(crash 旁路)。breach 概率是主门;breakeven 与 crash 是兜底。不要用 vol-speed AND 门控(表2 证明有害)。
- **Q:目标规模(size)?** **gamma-aware**:engage 后底仓 30%,按美元 gamma 线性升至满仓。这比任何固定分仓都好(表4:中和高 delta 风险;表3:DTE 越长优势越大)。
- **Q:持有时长(hold)/解除(unwind)?** hysteresis 双阈值(0.35 启 / 0.24 清)+ **min_hold 12min** + reduce 冷却 4min + **no-trade band 20%**。band 是抗 whipsaw 的单一最重要杠杆;减仓优先 maker。
- **Q:SOFT 初始分仓默认值?** 若启用 gamma-aware → 此参数基本无关紧要,取 0.40 作底即可。若坚持固定分仓(gamma OFF)→ 取 **0.40–0.50**(表7:stress 上更大更好,但 calm 上更大更 drag,取折中)。
- **Q:止盈暂停后的末 3h 更激进还是更克制?** **更克制(suppress_add)**:不新开对冲腿,但保留 crash-override 与 reduce/unwind。表8 显示它略优于 normal/aggressive,且与"放任到交割"设计自洽。

## 6. 失效模式（必须知道何时它会坑你）

1. **Whipsaw 单条最大值。** 即便 BEST 的**单条最差路径**(-2052)也比不对冲(-1631)更深 —— 对冲在一次急跌尖峰开仓、随即反转,会锁定一个不对冲本不会有的损失。注意:**CVaR5/CVaR1(最差 5%/1% 的平均)仍大幅更优**(平均尾部改善了),只是**绝对最差的那一条**可能更糟。band + min_hold + hysteresis 已把发生频率压到最低;残留是尾部保险的固有成本。
2. **IV<RV(卖便宜了波动)区间。** 对冲最差的环境。Current 在此**主动反噬**(表6);BEST 把大部分伤害中和,但无法变成明显的胜。**若 RV 持续 > IV,问题在定价不在对冲。** 可选:realized≫implied 时放宽 band 的阻尼项。
3. **小 notional 颗粒度。** 小额测试时(表9)固定 0.001 BTC / 0.1 tick 把成本占比抬到 ~30%、单位保护变差。**别用小测经济性外推全仓。**
4. **模型风险。** 若真实 BTC 跳跃比标定的 Merton/Regime 更肥更快,gamma-aware 可能低估真尾部;若更平静,BEST 也会过度对冲。**需用 FMZ 实盘 realized 路径重标定生成器。**
5. **近交割 gamma 爆炸。** 最后 1 小时美元-gamma → 极大,gamma-aware 会因微小波动推向满仓 → churn。suppress_add + 现有"末 3h 不主动止盈"部分缓解,但**真实 expiry-gamma 必须实盘验证。**

## 9. 决策建议

**采纳。** 把 gamma-aware + maker-reduce + hold-hedge 重构作为 CONFIG 门控的 **v3.2(`HEDGE_POLICY_V32`)**,默认关闭,保留 v3.1.3 作 fallback。

**理由综述:** 在 7 模型、跨 DTE / |delta| / W / IV-RV / notional 的全部 sweep 上,(a) 触发式对冲方向正确(保留哲学);(b) 当前 breach-prob 策略留有大量空间且在 IV<RV 反噬(需收紧);(c) gamma-aware sizing 是单一最大改进,maker-reduce 是免费成本胜,hold-hedge/band 直接解决你最担心的 whipsaw;(d) 三者叠加的 BEST 在 calm 上≈零成本保险、在 stress 上双赢。

**上线前必须做:**
1. 用 FMZ 实盘日志重标定路径生成器与成本/滑点(尤其近交割);
2. 先小步灰度,但**清楚小 notional 会高估成本占比、低估单位保护**(表9),用全仓位真实数据复核;
3. 默认 `final3h_mode=suppress_add`,但**确认 crash-override 在末 3h 仍生效**(代码已验证);
4. 把"绝不视缺失数据为 0""bundle 编译 ≠ 实盘通过"作为发布门槛。

**一句话给决策者:** 不是"要不要对冲",而是"把现有的恶化→对冲从 breach 概率升级为 gamma 感知,并用 band/min-hold/maker 把抖动成本压下去" —— 这在模型里把尾部保险的性价比从 Current 的中等提升到接近最优,且不触碰任何业务主链与交互面。
